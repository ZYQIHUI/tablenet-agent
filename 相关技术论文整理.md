# TableNet 相关技术论文整理

检索日期：2026-07-06  
主题论文：`TableNet: A Large-Scale Table Dataset with LLM-Driven Autonomous Generation`

## 1. 检索目标

围绕 TableNet 的贡献点，当前相关技术文献主要分为五条线索：

1. 表格结构识别（Table Structure Recognition, TSR）数据集与基准；
2. 深度学习 TSR 方法；
3. VLM/LLM 用于表格识别、表格理解或图像到 HTML；
4. LLM/程序化/合成数据生成；
5. 主动学习、CoreSet 和基于多样性的样本选择。

TableNet 的核心创新可以概括为：面向 TSR 的大规模、多样化、可控合成表格数据集；LLM 多智能体表格生成系统；以及基于多样性的主动学习样本选择。因此，下面文献按“和 TableNet 关系强弱”排序。

## 8. 可写入相关工作的段落结构建议

### 8.1 表格结构识别数据集

可先从 ICDAR、UNLV、Marmot 等早期数据集讲起，说明规模较小、场景受限；再介绍 PubTabNet、FinTabNet、TableBank、PubTables-1M 等大规模数据集。最后指出这些数据集多来自特定领域或固定采集流程，视觉样式、语言、结构复杂性和显式类型标签仍不足，因此 TableNet 通过可控合成补足多样性。

### 8.2 合成表格数据

SynthTabNet / TableFormer 是最重要的对照。可以强调已有合成数据通常依赖固定模板、特定分布或程序化规则，而 TableNet 的差异是将 LLM 用于主题、语义内容和多智能体调度，同时用规则工具约束 HTML 结构，有助于减少直接由 LLM 写 HTML 导致的结构不稳定。

### 8.3 VLM/LLM 表格结构识别

可以从 Donut、Pix2Struct、Nougat 等 OCR-free 文档理解模型讲到 Qwen2-VL、GPT-4V/4o 等通用多模态大模型。TableNet 的实验结果可用于论证：通用 VLM 具备一定视觉和语言能力，但对跨行、跨列、无线表格和彩色复杂布局的结构推理仍需要面向 TSR 的监督数据。

### 8.4 主动学习

先介绍主动学习的一般流程：初始化、查询、标注、训练、停止；再区分不确定性采样和多样性采样。TableNet 的合理性在于表格空间天然异构，行列数、跨单元格、表头方向、颜色、线型等因素构成复杂分布，因此 CoreSet/k-center 这类覆盖式选择方法更适合减少冗余样本。

## 11. 当前结论

TableNet 最强相关的文献不是单一方向，而是“TSR 数据集 + 合成表格数据 + 多模态模型 + 主动学习”的交叉。写相关工作时，建议不要只按时间线铺陈，而要突出 TableNet 解决的三个缺口：

1. 现有 TSR 数据集大多来源固定，结构、视觉和语义多样性不足；
2. 直接让 LLM 生成 HTML 容易结构不稳定，需要工具约束和多智能体分工；
3. 多样化数据池中存在大量冗余样本，基于多样性的主动学习能用更少样本达到相近或更好性能。
