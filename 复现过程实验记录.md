# TableNet 复现过程实验记录

## 1. 复现定位

本项目复现对象是 TableNet 论文中的核心机制，而不是直接复刻 44.5 万规模的完整数据集。

当前优先级是先完善论文中的**多智能体表格生成系统**，即用多 Agent 和规则工具生成可控、多样、有标注的表格图像。后续再逐步进入结构保真度实验、填充检查器实验、TSR 微调和主动学习验证。

当前阶段暂不作为主线推进：

- Qwen2-VL 全参数微调
- 论文级 445K 数据集生产
- PDF / Word 真实表格大规模爬取
- PubTabNet / FinTabNet / TableBank 等公开数据集完整对比
- HuggingFace 数据集发布

## 2. 原论文核心思想

TableNet 的关键不是让 LLM 直接端到端生成完整 HTML，而是把表格生成拆成两类问题：

- **结构确定性问题**：交给规则工具、结构验证器和备用 HTML 构造器。
- **语义开放性问题**：交给 LLM 或语义模板生成主题、表头和表体内容。

论文 Table 5 也说明，LLM 直接生成 HTML 容易出现标签错误、跨行跨列不一致、逻辑列数错位等问题；而 Agent Tool 能通过规则结构生成保持更高结构保真度。

因此，本复现的第一原则是：

```text
结构由工具保证稳定，语义由 Agent 或 LLM 增强，多样性由配置和增强策略控制。
```

## 3. 多智能体生成系统目标流程

论文式目标流程如下：

```text
用户配置
-> CoreAgent 编排流程
-> TopicAgent 生成领域主题
-> SchemaAgent 生成行列、表头层级、rowspan / colspan 结构
-> StyleAgent / CSS 工具生成视觉样式
-> HeaderAgent / BodyAgent 填充表头和表体内容
-> Validator / FillingChecker 检查结构、主题相关性和语义一致性
-> HtmlBuilder 输出 HTML 和 structure tokens
-> RendererTool 渲染图像
-> 输出图像、HTML、结构标注、单元格标注和视觉标签
```

当前实现已经具备最小可运行骨架，并已接入评分版 FillingChecker。整体仍以规则和模板为主，还不是完整的 LLM-driven multi-agent system。

## 4. 当前实现状态

当前代码中的生成流程位于：

```text
TableGeneration/agents/core_agent.py
```

当前实际流程：

```text
TableRequest 用户配置
-> TopicAgent 生成主题、行列规模和 simple / colored / lined 属性（支持 LLM 与规则 fallback）
-> SchemaAgent 生成表格结构
-> StyleAgent 生成 CSS 样式
-> HeaderAgent 填充标题和表头
-> BodyAgent 根据表头语义填充表体（支持 LLM 与规则 fallback）
-> ValidatorAgent 检查结构合法性
-> FillingChecker.evaluate() 计算填充质量分、警告和错误
-> HtmlBuilder 输出 HTML 和 structure_tokens
-> RendererTool 渲染图像并输出 gt.txt / meta.jsonl
```

当前已覆盖：

- CoreAgent 编排
- TopicAgent 主题与属性规划，支持 `.env` 配置和 OpenAI-compatible LLM 接口
- SchemaAgent 基础结构生成
- HeaderAgent / BodyAgent 内容填充，支持规则 fallback 和可选 LLM 入口
- BodyAgent 已能根据表头关键词生成数值、百分比、负责人、状态等字段值，并做基础归一化
- ValidatorAgent 基础结构检查
- FillingChecker 基础语义检查与质量评分
- StyleAgent 基础 CSS 样式
- HtmlBuilder HTML 构造
- RendererTool Selenium 渲染
- gt.txt 和 meta.jsonl 基础标注输出
- 目录已重构为 `sub_agents/` 与 `tools/` 两层，`core_agent.py` 保持在根目录作为大脑入口
- 原始 `GenerateTable` 与 `Table` 继续复用，负责最终渲染与 PP-Structure 风格标注

当前主要缺口：

- complex 表格结构仍偏简单，rowspan / colspan / 多级表头还可以继续增强
- 还没有在 `CoreAgent` 中加入失败后的局部重试或重建策略
- 视觉样式还没有覆盖论文中的字体、水印、背景扰动等更丰富因素
- 标注输出还没有达到 S / C / H / V 完整体系
- 还没有 5 个原始变体 + 4 个增强变体的数据增强流程
- 还没有两级记忆机制
- 还没有主动学习和 TSR 训练脚本

## 6. Agent 职责边界

为避免“Agent 数量变多但职责不清”，后续实现应坚持以下边界：

| Agent / 工具 | 输入 | 输出 | 责任边界 |
|---|---|---|---|
| CoreAgent | TableRequest | AgentTable 或失败信息 | 只负责编排、重试和反馈，不直接生成结构或内容 |
| TopicAgent | domain、language、属性约束 | TablePlan | 负责主题、领域、语言、行列规模和高层属性 |
| SchemaAgent | TablePlan | TableSchema | 只负责结构，不负责文本内容 |
| StyleAgent | TablePlan | TableStyle | 只负责视觉属性和 CSS，不改变结构 |
| HeaderAgent | TableSchema、TablePlan | 填充表头后的 TableSchema | 只填 title / header / th |
| BodyAgent | TableSchema、TablePlan | 填充表体后的 TableSchema | 只填 body / td |
| ValidatorAgent | TableSchema | 结构是否合法、错误列表 | 只判断结构合法性 |
| FillingChecker | TableSchema、TablePlan | FillingCheckReport 或 (bool, errors) | 判断主题相关性、字段类型和语义一致性，并给出评分 |
| HtmlBuilder | plan、schema、style | HTML、structure_tokens、id_count | 只做 HTML 构造 |
| RendererTool | AgentTable | 图片、HTML、gt、metadata | 只做渲染和标注输出 |

## 9. 填充检查器实验

FillingChecker 需要评估三个维度：

| 维度 | 方法 | 说明 |
|---|---|---|
| 结构正确性 | 启发式 | HTML 标签、逻辑列数、span 合法性 |
| 主题相关性 | LLM 或规则排序 | 内容是否围绕指定 topic |
| 语义一致性 | LLM 或规则排序 | 表头-表体是否匹配，内容是否自洽 |

当前实现已经从“布尔检查器”升级为“评分检查器”：

- `evaluate()` 返回 `FillingCheckReport`
- 报告里包含整体分数 `score`
- 还包含 `title_score / header_score / body_score`
- 同时输出 `errors / warnings`
- 也记录 `column_scores / cell_scores`
- `check()` 保留旧接口，便于兼容主流程

当前已实现的规则与评分逻辑：

- body 单元格不能为空
- 含“率、同比、进度”等表头的列应生成百分比
- 含“数、收入、预算、支出、利润、完成率、评分”等表头的列应生成数值或百分比
- complex 表格的标题单元格应包含规划 topic，或至少与 topic 有明显 token overlap
- 对 entity / status / numeric / percent 类型分别给不同评分
- 对明显通用占位词、空值和弱匹配内容给出 warning 或 error

下一步需要补齐：

- 更强的表头-表体一致性判断
- 更细的主题相关性排序
- 状态、日期、金额、编号等更多字段类型检查
- 失败后的局部重填或 fallback
- 把评分结果真正接到重试机制里，而不是只在报错时展示

总分策略参考论文：

```text
overall_score = min(structure_score, topic_score, semantic_score)
```

### 9.1 人工相关性实验

实验流程：

1. 随机选 100 个原始表格。
2. 每个表格构造 3 类扰动：结构扰动、主题扰动、语义扰动。
3. 每组形成 4 个版本：原表 + 3 个扰动表。
4. 人工按质量排序。
5. FillingChecker 也输出排序。
6. 计算 Spearman、Pearson、Kendall。

目标：

| 指标 | 目标 |
|---|---:|
| Spearman | > 0.70 |
| Pearson | > 0.70 |
| Kendall | > 0.60 |

### 9.2 当前实现进展

截至目前，`FillingChecker` 已经完成以下工作：

- 从单纯返回 `bool` 转为返回结构化报告
- 支持 title / header / body 三部分分项评分
- 支持错误、警告和列级别、单元格级别反馈
- 与 `CoreAgent` 完成对接
- 已通过冒烟测试，`CoreAgent().generate(...)` 可正常生成表格并得到高分报告

| 类型 | 数量 | 操作 |

## 14. 风险与处理

| 风险 | 影响 | 处理 |
|---|---|---|
| LLM 内容跑题 | 检查器相关性低 | 加强 topic prompt，加入实体约束 |
| HTML 结构错误 | 数据不可训练 | 规则生成结构，LLM 只填内容 |
| 渲染图像不稳定 | 训练噪声大 | 固定浏览器 viewport 和 CSS reset |
| 训练资源不足 | TSR 微调中断 | 使用 LoRA、减少图像尺寸或转 4090 |
| CoreSet 特征提取困难 | 主动学习卡住 | 使用 CLIP / SigLIP 作为备用特征 |
| TEDS 实现不一致 | 结果不可比 | 同时报告 structure-only TEDS 和 full TEDS |
| 人工排序成本高 | Table 2 复现受阻 | 先做 50 组扰动，再扩展到 100 组 |
